## A. 課金・プラン・お試し（🔴 課金開始前に必須）

- **A1. 無料お試し1ヶ月→自動課金の実装**
  - 論点: Square のトライアル設定と、期限到来→自動課金/降格の流れ。
  - 未確定: Square 側サブスクのトライアル1ヶ月設定の具体手順 ／ 期限切れ判定を Square の Webhook 任せにするか、Netlify Scheduled Function で `trial 終了日` を日次チェックして降格するか ／ ダッシュボードに「お試し残り○日」を出すか。

- **A2. 解約・ダウングレード時の「超過データ」の扱い**（最重要・複数機能に波及）
  - 論点: 有料→無料に下がったとき、無料上限を超える既存データをどうするか。
  - 未確定: **予約ページ3〜5個目**（無料は2まで）→ 凍結（編集不可・非公開）／削除／アーカイブのどれか ／ **事前アンケート3〜5問目**（無料は2まで）→ 削除／非表示のみ／ゲストには何問見せるか ／ **再昇格時の復元**可否。
  - 備考: Cat Key 強制降格（B4）でも同じ判断が必要。

- **A3. 支払い失敗時の挙動**
  - 論点: 課金失敗イベント受信時の扱い。
  - 未確定: 即時 `free` 降格か、猶予日数を設けるか ／ ユーザー通知の有無。

- **A4. Square Webhook の網羅性・冪等性**
  - 論点: 受信イベント種別と重複/喪失対策。
  - 未確定: subscription/payment/cancel/fail 等のどのイベントで何をするかの確定 ／ 重複配信時の冪等性（処理済み記録）／ 署名検証（`SQUARE_WEBHOOK_SHARED_SECRET`）の運用。

- **A5. 税・インボイス・返金**
  - 論点: ¥980（税込）の税運用と適格請求書。
  - 未確定: Square 設定で消費税が正しく内包/表示されるか ／ Square 発行のインボイスに登録番号（T1030005019164）が入るか ／ 返金時の手続き・インボイス修正方針（特商法表記は「返金なし」だが例外運用の有無）。

- **A6. 上位プラン（マスター/法人・エンタープライズ）**（⚪ 将来）
  - 論点: 2段階（無料/Pro）の先の料金体系。
  - 未確定: マスタープラン（議事録、I参照）の価格・含む機能・時期 ／ 法人プランの内容（人数か機能か）・価格帯。
