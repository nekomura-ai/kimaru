## 4. 運用・公開（5分）

### ★ メール送信方式（今日決める）
**背景**: メールは予約確定・ホスト通知・キャンセル/変更・リマインダー・誕生日に使う。`@gmail.com` から送るには Google経由しかなく、**アプリパスワード＝2段階認証が必須／OAuth＝2FA不要だが設定が複雑**。**「2段階認証は避けたい」意向**を踏まえ、下の3案から選ぶ。

| 案 | お金 | 2FA | ドメイン | 手間 | 状態 |
|---|---|---|---|---|---|
| ① 独自ドメイン+Resend | 年¥1,500前後 | 不要 | 要 | 小（Resendキー作成済・実装済） | env入替のみ |
| ② Gmail API + OAuth | **¥0** | 不要 | 不要 | 中（OAuth設定＋本番公開でトークン無期限化） | 要実装 |
| ③ Gmail SMTP（アプリパスワード） | ¥0 | **必要** | 不要 | 最小（実装済 PR#177） | 2FA ONが前提 |

- 選択: [ ] ① ドメイン+Resend / [ ] ② Gmail API(OAuth) / [ ] ③ Gmail SMTP(2FA)
- 補足: ①②③いずれも無料Gmail/Resend無料枠の範囲（数百通/日）で立ち上げ期は十分。②は「テスト段階だとトークン7日失効→**本番公開で回避**」に注意。③は最も簡単だが2FAが要る。
