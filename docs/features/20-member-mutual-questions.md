# 20. キマル会員同士の相互質問（双方向）

[← 機能一覧に戻る](./README.md)

- ステータス: ❌ 未実装（打ち合わせ 2026-06-03 で新規決定）
- 対象プラン: 共通（会員同士のとき有効）
- 仕様: [`../spec.md`](../spec.md) 主要機能 11

## 概要

予約する側だけが情報を出す片方向ではなく、ログイン済みのキマル会員同士なら **予約された側からも相手へ質問を投げられる** ようにする。

## 仕様詳細

- 「○○さんから、この日程で予約が来ました」という通知に対し、受け取った側からも質問事項を返せる。
- プロフィールはお互いに見える状態にする（既に入力済みの情報を、欲しい人に渡せないのは避ける）。
- 双方が会員のときに、より関係構築が進む体験を作る（調整だけで終わるドライな体験にしない）。

## 背景（打ち合わせ）

- 現状のキマルは「予約する側が情報を出す」攻め偏重（＝ポジション・先手が有利）。
- せっかく両者が会員（＝両者課金）なら、相互に質問でき、Zoom 中に「一緒だと良いね」となる体験が望ましい。

## 関連ファイル（想定）

- `public/booking.html` / `book.js` — 予約成立時に「相手への質問」導線を追加
- ログイン状態（`/api/me`）で会員同士かを判定
- DB: `questionnaire_*` の双方向化、または新規の相互質問テーブル

## 残タスク

- 会員同士判定（予約者・被予約者ともログイン会員か）。
- 被予約者 → 予約者への質問送信・回答の保存と表示。
- プロフィール相互閲覧（[17](./17-profile.md)）との連携。
