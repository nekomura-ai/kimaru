# 16. 生年月日の占いベース相手分析（誕生日メールは廃止）

[← 機能一覧に戻る](./README.md)

- ステータス: ✅ ルールベース算出を実装（算命学=年柱五行＋数秘術=ライフパス・#20）。AI/日柱の精密化は将来。**誕生日メール自動送信は廃止（2026-06-09・#180）**
- 対象プラン: 有料（Pro）
- 仕様: [`../spec.md`](../spec.md) / 決定: [`../open-decisions.md`](../open-decisions.md) 17・18

## 概要

生年月日入力の目的は **「占いベースの相手分析」**。相手を分析し、**アポイント時に気をつけること・注意点・相手に合わせた接し方**を発行者に提示する（Proの価値）。
※**誕生日お祝いメールの自動送信は廃止**（関係悪化時の逆効果・プライバシー懸念のため。2026-06-09 決定）。生年月日の収集と占い分析は継続。

## 仕様（2026-06-09 決定）

- **採用する占い体系: 算命学 ＋ 数秘術**（誕生日のみで算出可能なもの。四柱推命は時刻・出生地が要るため見送り、動物占いは算命学系として包含）。
- **まずルールベースで実装**（計算式で決定的に算出＝API費用なし・無料運用）。有料会員が増えキャッシュが出たら AI/上位プラン化を検討。
- 見せ方は **断定せず「こういう傾向があります／理論的な方が通るかも」** 等。**相手管理画面＋リマインダーの両方**に出す。
- 生年月日は **「非公開」を選べる**（非公開時は分析・表示をマスク。実装済）。
- ルールベース実装済み（#20）。**算命学＝年柱の五行**（立春補正つき・年干→木火土金水）＋**数秘術＝ライフパスナンバー**（生年月日の数字和を1桁に還元、11/22/33はマスター保持）で、相手の傾向・接し方・避けたい入り方を決定的に算出。

## 現状の実装

- **生年月日インサイト（算命学年柱＋数秘術・#20）**: `public/booking-week.js` / `public/app.js` の `buildRelationshipProfile` が、生年月日から **年柱の五行＋数秘ライフパス** を算出し、予約時に `bookings.relationship_profile` へ保存して相手管理一覧で表示。生年月日非公開時はマスク。
  - 精密化の余地: **算命学の日柱（日干支）算出**（要・暦データ）と AI による文章生成は将来（プレミアムのAIアシスト [18](./18-ai-assist.md) と連携余地）。
- **誕生日メール（廃止済み・#180）**: 自動送信機能（`birthday-mails.js` / `birthday-scheduled.js` / `netlify.toml` のスケジュール）は**削除済み**。`birthday_message_deliveries` テーブルと `birthday_message_opt_in` 列は無害な遺物として残置（参照なし）。

## 関連ファイル

- `public/app.js` / `public/booking-week.js` — `buildRelationshipProfile`（生年月日インサイト・占い分析の土台）
- DB: `bookings`（`visitor_birth_date` / `relationship_profile`）。`birthday_message_opt_in` / `birthday_message_deliveries` は廃止機能の遺物（未使用）。
