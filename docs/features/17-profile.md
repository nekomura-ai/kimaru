# 17. プロフィールシート

[← 機能一覧に戻る](./README.md)

- ステータス: ⚠️ 部分実装（サーバ保存API実装済み。高度版エディタは未）
- 対象プラン: 共通
- 仕様: [`../spec.md`](../spec.md) DB 設計 `profiles`

## 概要

発行者が自分のプロフィール（自己紹介・強み等）を整え、リマインド（[21](./21-reminder.md)）や AI アシスト（[18](./18-ai-assist.md)）に渡す。

## 仕様（打ち合わせ 2026-06-03 / 高度版）

有料版では自由度の高いプロフィール編集ができる（参考: セルプロのようなプロフィール表示サービス）。

- 文字色・太字・見出し・枠・画像など、レイアウトを自由に編集できる。
- 共有は「プロフィールはこちら」形式のハイパーリンク（生 URL は貼らない＝押しやすさ・安心感）。Google カレンダー上にも同形式で出せると望ましい。
- リマインダー（[21](./21-reminder.md)）に載せるプロフィールの中身にもなる。
- キマル会員同士はプロフィールを相互に閲覧できる（[20](./20-member-mutual-questions.md)）。
- 派生として「プロフィールを代わりに作成する」有料サービス等の展開余地。

## 現状の実装

- `profile.html` でプロフィールを入力・編集。ログイン確認に `/api/me` を使用。
- **保存先は localStorage**（`profileKey`）。サーバ DB（`profiles` テーブル）へは未保存。
- 整えたプロフィールは同一ブラウザの AI アシスト（[18](./18-ai-assist.md)）が参照。

## 関連ファイル

- `public/profile.html` — 入力 UI・localStorage 保存
- DB: `profiles`（仕様。現状未使用）

## 残タスク

- `profiles` テーブルへのサーバ保存 API（端末間共有・永続化）。
- プロフィール付きリマインドメール（22分前）への反映（[21](./21-reminder.md)）。
- **高度版エディタ**（色・太字・見出し・枠・画像）と「プロフィールはこちら」リンク生成。有料機能として。
- 会員同士のプロフィール相互閲覧（[20](./20-member-mutual-questions.md)）。
