# 15. 多言語対応（i18n）

[← 機能一覧に戻る](./README.md)

- ステータス: ✅ 実装済（対応3言語）
- 対象プラン: 共通
- 仕様: [`../spec.md`](../spec.md) UI 方針（一般ユーザー目線）

## 概要

UI を複数言語で表示する。初期表示は日本語優先。全ページのヘッダーに言語選択プルダウンを設置し、選択で即時切替・選択を localStorage に保存する。

## 仕様詳細

- 一般ユーザー目線の表現（ポーカー専用表現は使わない）。
- 軽量な構成を維持。
- 対応言語は **日本語 / English / 繁體中文 の3言語**（方針として3言語に整理）。

## 現状の実装

- ✅ 対応 3 言語（ja / en / zh-TW）の翻訳辞書を完備。
- ✅ 全ページ（index / signup / booking / admin）に言語選択プルダウンを設置（ヘッダー）。
- ✅ 全ページで `i18n.js` を読み込み、`data-i18n` / `data-i18n-placeholder` / `data-i18n-title` で翻訳。
- ✅ 動的文言（予約メッセージ、管理ダッシュボードの状態表示など）も `t()` 経由で翻訳。
- ✅ 言語切替時、管理画面の動的コンテンツは `kimaru:languagechange` イベントで再描画。
- 言語選択は localStorage → 既定（ja）→ ブラウザ言語 → 英語 の順で決定。

## 関連ファイル

- `public/i18n.js` — 翻訳辞書（3言語）・言語切替・`setLanguage` の `kimaru:languagechange` 発火
- `public/app.js` — 動的文言の `t()` 化・言語変更時の再描画
- 各 `public/*.html` — `data-i18n` 属性・`<select data-language-select>`
- `public/styles.css` — `.lang-select` スタイル

## 残タスク

- 新規追加機能（[03](./03-duration.md)〜[14](./14-customer-management.md)）を実装する際、その文言キーを 3 言語ぶん辞書へ追加。
- 予約スロットの日付ラベルは言語切替時にリロードまで旧ロケール表記が残る（再描画は管理画面のみ実装）。必要なら予約画面にも再描画を追加。
- 表現がポーカー寄りでないか継続チェック。
