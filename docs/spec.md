# キマル（Kimaru）仕様書

最終更新: 2026-06-03（打ち合わせ反映）

## 目的

無料範囲で運用できる日程調整ツール「キマル」を完成させる。
フロントは Vercel、DB は Supabase を使う。

一般ユーザー目線で作る。ポーカー専用の表現は使わない。
猫の集会は「招待コード特典」としての運用で進める。
HTML / JS / CSS のシンプル構成で、高速な web サービスを目指す。

## 技術構成

| 領域 | 採用 |
|---|---|
| Frontend / Host | 当面 Netlify で運用（Vercel は将来用に維持。決定 2026-06-03） |
| Database | Supabase |
| Auth | Google ソーシャルログイン ＋ メール+パスワード（決定 2026-06-03） |
| Calendar | Google Calendar API |
| Meeting | Google Meet 自動発行を優先実装 |
| Zoom | 将来的に実装できる設計にしておく |

## 主要機能

### 1. Google カレンダー連携

- 発行者の Google カレンダー予定を取得
- 既存予定を避けて空き時間のみ表示
- 予約確定後、Google カレンダーに予定を自動作成

### 2. 予約時間設定

発行者が以下から選択できる。

- 30分
- 45分
- 60分

> **複数の予約ページ（日程調整URL）**: 設定の異なる予約ページを複数保存できる。保存数は **無料2つ / 有料・猫メンバー5つ** まで（[features/24](./features/24-multiple-booking-pages.md)）。1ページ＝時間・バッファ・受付期間・開催方法・受付時間・アンケートの1セット。

### 3. 前後バッファ設定

発行者が設定できる。

- 前バッファ: なし / 15分 / 30分
- 後バッファ: なし / 15分 / 30分

### 4. 予約受付期間

発行者が選択できる。

- 1ヶ月先まで
- 2ヶ月先まで
- 3ヶ月先まで
- 6ヶ月先まで

| プラン | 公開可能範囲 |
|---|---|
| 無料版 | 2ヶ月先まで |
| 有料版 | 6ヶ月先まで |

> 打ち合わせ（2026-06-03）で無料版の上限を **3ヶ月 → 2ヶ月** に変更（有料は6ヶ月のまま、「2と6（ニャンニャン）」）。3ヶ月先まで埋まるのは相当なヘビーユーザーで、その層は有料対象という判断。
> ※実装は現状 3ヶ月のため、2ヶ月への変更が必要（[features/05](./features/05-booking-range.md)）。

### 5. 開催方法

予約ページ作成時に、発行者側が開催方法を選択できる。

選択肢:

- 対面
- Google Meet 自動発行
- Zoom 自動発行 ※将来対応
- 電話
- カスタム URL
- 後で連絡

各選択肢の挙動:

- **Google Meet を選択した場合**
  - 予約確定時に Google Meet リンクを自動発行
  - 予約完了メールに Meet リンクを記載
  - Google カレンダー予定にも Meet リンクを反映
- **対面を選択した場合**: 住所入力欄を表示
- **電話を選択した場合**: 電話番号または案内文を設定可能にする
- **カスタム URL を選択した場合**: 発行者が任意 URL を入力できる

### 6. 事前アンケート

無料版と有料版で質問数を分ける。

**無料版**

- 最大 2 問まで
- 初期質問:
  1. 今回お話したい内容
  2. 今、実現したい夢や目標は何ですか？

**有料版**

- 最大 5 問まで
- 質問内容を自由に編集可能
- 必須 / 任意を設定可能

> 打ち合わせ（2026-06-03）: 質問数は **無料2問・有料5問** で確定。当初20問案もあったが「面接のようで重い・回答途中で離脱する」ため、有料でも5問が上限。「有料にすると質問を選べる」という体験差を見せる狙い。
> ※生年月日（アンケートとは別枠で取得）は **「非公開」を選べる**ようにする。入力したくない相手のための選択肢。

推奨質問例:

- 今回お話したい内容
- 今、実現したい夢や目標は何ですか？
- 最近挑戦していることはありますか？
- 応援してほしいことはありますか？
- 趣味や好きなことはありますか？

**回答形式**

- まず **自由入力** で実装（リリース優先・最短で出せる）。
- 将来的に **選択式** にも対応し、Google フォームのように「選択肢を発行者が作る」「自由入力／選択式のどちらも選べる」状態を目指す。
- ゆくゆくは「よく使われる質問トップ5」などのデータ収集も行う（初期実装はしない）。

### 7. 有料版機能

有料版では以下を実装、または実装できる設計にする。

- 6ヶ月先まで予約公開
- 予約ページ（日程調整URL）5つまで保存
- 複数名での日程調整
- 事前アンケート最大 5 問
- 顧客管理
- 予約履歴管理
- 高度なプロフィールシート（色・太字・見出し・画像・枠などの自由編集）→ [主要機能 12](#12-プロフィールシート高度版)
- 生年月日からの占い的インサイト（相手に合わせたアプローチ方法・仲良くなる手段の提示）
- AI 要約機能 ※将来対応
- Zoom 自動発行 ※将来対応（無料・有料とも提供。有料限定ではない）

### 8. 無料版機能

無料版では以下を使える。

- Google カレンダー連携
- 1 対 1 予約
- 予約ページ（日程調整URL）2つまで保存
- 2ヶ月先まで公開
- 30分 / 45分 / 60分予約
- 前後バッファ設定
- 事前アンケート最大 2 問
- Google Meet 自動発行
- Zoom 自動発行 ※将来対応（無料・有料とも）
- 予約完了メール

### 9. 招待コード（Cat Key / 猫の鍵）

猫の集会メンバー用の招待コード入力欄を作る。

- 招待コード: `Neko20240222`
- このコードを入力したユーザーは、有料版機能を無料で使えるようにする。

**不正利用対策（打ち合わせ 2026-06-03）**

無料利用が口コミで広がり、本来の対象外まで無料利用が増えるのを防ぐ。

- **承認制にする**: Cat Key を入力しても即時に無料付与せず、運営が承認した場合のみ有効化する。これで不正利用をほぼ防げる。
- **強制ダウングレード**: 不正・退会済みなどが判明した場合、運営がそのアカウントを強制的に有料 → 無料へ降格できる。
- 「承認する／降格する」の両方を運営側で操作できるようにする（運営者管理画面）。
- **将来**: Cat Key を定期更新（自動ローテーション）し、猫の集会のグループチャットから最新キーを取得する運用にする。退会者は旧キーが無効になるため、より確実に防げる。

### 10. リマインダー（22分前 / プロフィール付き）

予約した打ち合わせの **22分前** に、相手へリマインダーを送る。

- タイミングは **22分前**（「ニャンニャン前」）。当初15分前案だったが、ネット遅延・準備の余裕を考え22分に決定。15分あれば相手のプロフィールに目を通せる、というのが狙い。
- リマインダーには **打ち合わせ相手のプロフィール情報** を載せる。事前に相手を知ることで距離が縮まり、当日の会話につながる。
- プロフィールは生 URL ではなく「プロフィールはこちら」のようなハイパーリンク形式で見せる（押しやすさ重視）。Google カレンダー上でも同様に出せると望ましい。

### 11. キマル会員同士の相互質問（双方向）

予約する側だけが情報を出す片方向ではなく、ログイン済みのキマル会員同士なら **予約された側からも相手へ質問を投げられる** ようにする。

- 「○○さんから予約が来ました」という通知に対して、受け取った側からも質問事項を返せる。
- プロフィールはお互いに見える状態にする（既に入力済みの情報を、欲しい人に渡せないのは避ける）。
- 双方が会員のときに、より関係構築が進む体験を作る。

### 12. プロフィールシート（高度版）

有料版では、自由度の高いプロフィール編集ができる（参考: セルプロのようなプロフィール表示サービス）。

- 文字色・太字・見出し・枠・画像など、レイアウトを自由に編集できる。
- 「プロフィールはこちら」形式のハイパーリンクで共有（生 URL を貼らない）。
- リマインダー（[主要機能 10](#10-リマインダー22分前--プロフィール付き)）に載せるプロフィールの中身にもなる。
- 派生として「プロフィールを代わりに作成する」有料サービスなどの展開余地もある。

### 13. お試し期間（無料トライアル）

有料プランに **無料お試し期間（1ヶ月）** を設ける。

- **登録時にカード登録あり**。無料期間（1ヶ月）終了後、自動で有料（¥980/月）へ移行する。
- 途中解約すれば課金されない。Square のサブスク（トライアル）機能で実現。
- まず使ってもらい「良い」と感じてもらえれば継続につながる、という導線。

### 14. 将来構想

すぐには実装しないが、設計・議論として残す。

- **AI 検索 / 顧客検索**: 登録ユーザーを属性（趣味・職種など）で検索し、こちらからアプローチできる機能。ビッグデータとしての質問傾向の活用も含む。
- **マスタープラン**: Google Meet の録音 → 議事録自動生成（工数大のため後回し）。プロプランの上位。自前実装ではなく既存の議事録AI（tl;dv / Fireflies / Circleback / Meet純正API）との連携が現実的 → 調査メモ [features/23](./features/23-meeting-minutes.md)。
- **法人 / エンタープライズプラン**: 組織向け。価格は月1万円〜も想定。Deeps のような紹介・マッチング型課金（段階課金）も参考。

## 16. UI 方針

- 一般ユーザー目線で作る
- ポーカー専用の表現は使わない
- 猫の集会は招待コード特典としての運用で進める
- HTML / JS / CSS のシンプル構成で高速な web サービスを目指す

### デザイン方針（打ち合わせ 2026-06-03）

- デザイン案を確認。「予定が決まる」タイトルが**青**の案（構成が分かりやすい）が好評。グローバル感・王道感もこの青がよいという意見。
- **構成はこの案で固定し、色は変更可**（青のままでも別色でも可）。
- **黒背景（ダークSaaS系）と HQ／コマンドライン風はナシ**。カクカク系・一般系・優しい色味はいずれも可。
- 最終的な配色・デザインはタケダ氏にも相談し、ユーザー（女性目線含む）視点で決める。
- 候補は `public/designs.html`（`landing.html` / `landing2`〜`landing7`）。
